# Vegas Blitz Deterministic Edge Cases

Rules:
- Expected payout and feature values are not hand-authored here.
- EXPECTED_RESULT is always execution through `generation/tests/run_forced_window.cpp`.
- OBSERVED_RESULT values must be copied from `validation/oracle/edge_results.json`.

  [LV2,  LV3, LV4, LV5, LV6, HV1],
  [HV2,  HV3, HV4, HV5, LV1, SCAT],
  [LV3,  LV4, LV5, LV6, HV1, HV2]
]

ACTIVE_WINDOW:
SAME_AS_FULL_WINDOW

INITIAL_STATE:
mode=BASE; bet_fixed_coins=20

EXPECTED_RESULT:
EXECUTE_WITH_FORCED_WINDOW_RUNNER

OBSERVED_RESULT:
{
  "ways_win": 10,
  "scatter_win": 0,
  "scatter_count": 2,
  "free_spin_award": 0,
  "blitz_spin_award": 0,
  "window_shape_valid": true,
  "non_negative_fields": true
}

AUDIT_STATUS:
PASS

FUNCTIONS_UNDER_TEST:
waysWinCalculation; freeSpinAward; blitzSpinAward

INVARIANTS_TO_CHECK:
Window shape remains [4x6]; scatter count is non-negative; awards are observed from core; feature threshold behavior is structural only.

NOTES:
This case does not assert a payout expectation by manual calculation.
