# Tester Escalation Report

## Blockers

None.

## Ambiguities

[AMBIGUOUS] Free Spins SCAT evaluation rule conflicts. `Game Rules` says all symbols pay left-to-right except SCAT and the paytable marks SCAT as "pays anywhere"; `Implementation` says Free Spins calculate left-to-right Scatter wins. Simulator can proceed only if this is modeled as an explicit selectable rule or clarified before final validation.

[AMBIGUOUS] 4096-ways duplicate counting is not formally specified. Source states "Each symbol can be used once per winning combination"; QA notes show ways multiplication examples, but no full formal rule for duplicate symbol/wild counting across a 6x4 ways window.

[AMBIGUOUS] Buy Bonus trigger display count conflicts in QA workbook. Math model and planner map Buy Bonus options to 4, 5, and 6 SCAT; QA rows 403-405 state "6 Scatter symbols should land" for all three options, while rows 411 and 414-425 expect 4/5/6 SCAT behavior by option.

[AMBIGUOUS] Buy Bonus Free Spins wild multiplier scope has conflicting wording. General Free Spins rules apply all visible WILD multipliers regardless of participation; Buy Bonus notes mention multiplying by all WILDs "till that reel." Use the planner's all-visible rule only as an explicit assumption.

## Warnings

[WARNING] Coin denomination and complete bet ladder are not specified in the math/rule source. Core simulator can use xBet/paytable units and the documented 20 fixed coin game cost, but currency-facing payout tests require external configuration.

[WARNING] Buy Bonus is documented in the math workbook and QA workbook, but UKGC rules/test scenarios indicate it is not available in UKGC mode. Treat Buy Bonus availability as jurisdiction/configuration-controlled, not always enabled.

[WARNING] QA workbook contains mostly UI/game-log/force-server scenarios and does not provide complete deterministic 6x4 simulator windows for all core mechanics. Oracle windows generated by Tester are derived from documented math rules and must be reviewed.

[WARNING] RA workbook asks whether base-game Coin symbols can appear anywhere and the maximum number on one reel; no answer is recorded. Implementation sheet permits additional COR placement anywhere except COLLECT/WILD/SCAT, so use that rule for injected additional CORs.

## Simulator Trace Verification

[WARNING] `trace_ref.txt` uses `SCATTER` while Planner outputs normalize the symbol as `SCAT`. Simulator schema aliases `SCATTER` to `SCAT` for trace compatibility, but downstream implementation should use one canonical symbol name.

[WARNING] `trace_ref.txt` omits `LV6` from its standard symbol list and sets `NO_OF_SYMBOLS = 8` / `SYMBOL_COUNT = 8`; Planner outputs define standard symbols `HV1`-`HV5` and `LV1`-`LV6` plus `SCAT`, `WILD`, `COR`, and `COLLECT`. Corrected trace uses `STANDARD_SYMBOL_COUNT = 11` and keeps special symbols outside ways iteration.

[WARNING] `trace_ref.txt` names reelsets `BG1_Reels`, `BG2_Reels`, and `FG1_Reels`; Planner outputs provide the same logical reelsets as `base_B1`, `base_B2`, and `free_F1`. Corrected trace maps these names explicitly.

[WARNING] `trace_ref.txt` declares reel size vectors as shape `[5]` in trace lines, but Planner outputs and Vegas Blitz layout require six reel sizes. Corrected trace uses `BG1_ReelSize`, `BG2_ReelSize`, and `FG1_ReelSize` with shape `[6]`.

[WARNING] `trace_ref.txt` describes the pay window as `NO_OF_ROWS x NO_OF_REELS` and shape `[4x6]`; Planner outputs describe a 6 reel x 4 row slot. Corrected trace standardizes storage as row-major `PayWindow` shape `[4x6]` while preserving the game layout as six reels by four rows.

[WARNING] `trace_ref.txt` references `PayTable` shape `[9x6]`, but Planner outputs define Vegas Blitz paytable entries for 11 standard symbols plus `SCAT`, with paying counts 2-6 for eligible symbols. Corrected trace uses a normalized symbol-keyed paytable with matrix view `[12x7]` so count indexes 0-6 can be represented without inventing pay values.

[WARNING] `trace_ref.txt` probability table names are present in Planner outputs under different names: `ReelsetWeights` maps to `base_reelset_selection_weights`, `corTrigger` to `base_additional_cor_trigger_weights`, `numCorWeight` to `base_additional_cor_count_weights`, `wildMult` to Free Spins `wild_multiplier_weights`, `freeGamenumCollect` to Blitz `collect_count_weights`, `freeGameCOROccur` to Blitz `cor_occurrence_weights`, `goodCORTable`/`badCORTable`/`CORValues` to `blitz_cor_value_weights`.

[WARNING] `trace_ref.txt` labels the Blitz Spins feature as `FreeGameFeature_2` and its probability tables as `freeGame*`; Planner outputs define this as Blitz Spins, not Free Spins. Corrected trace preserves the trace function alias but marks it as Blitz Spins.

[WARNING] `trace_ref.txt` base `runOneSpin` call path omits Collect resolution after ways/scatter evaluation. Planner and validation reports require Collect resolution before feature trigger handling. Corrected trace inserts `resolveCollect` after `waysWinCalculation`.
