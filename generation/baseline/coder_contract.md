# Vegas Blitz Coder Contract

Generated by the Simulator refinement pass. This is the canonical Coder-stage contract. The Coder should implement from this file without consulting original game documents.

Non-goals: do not write or alter game mathematics here; do not invent missing mechanics; do not treat `FreeGameFeature_2` as Free Spins. It is the trace-compatible alias for Blitz Spins.

## PayTable Contract

The paytable is symbol-keyed for:

```text
HV1, HV2, HV3, HV4, HV5,
LV1, LV2, LV3, LV4, LV5, LV6,
SCAT
```

Matrix view shape is `[12x7]`. Rows are the 12 payable symbols in the order above. Columns are symbol counts `0, 1, 2, 3, 4, 5, 6`. Missing counts pay `0`.

`WILD`, `COR`, and `COLLECT` do not have direct paytable rows. `WILD` substitutes for standard symbols only. It does not substitute for `SCAT`, `COR`, or `COLLECT`.

| Symbol | 0 | 1 | 2 | 3 | 4 | 5 | 6 |
|---|---:|---:|---:|---:|---:|---:|---:|
| `HV1` | 0 | 0 | 20 | 30 | 40 | 50 | 100 |
| `HV2` | 0 | 0 | 10 | 20 | 30 | 40 | 50 |
| `HV3` | 0 | 0 | 10 | 20 | 30 | 40 | 50 |
| `HV4` | 0 | 0 | 4 | 16 | 24 | 30 | 40 |
| `HV5` | 0 | 0 | 4 | 16 | 24 | 30 | 40 |
| `LV1` | 0 | 0 | 0 | 10 | 20 | 24 | 30 |
| `LV2` | 0 | 0 | 0 | 10 | 20 | 24 | 30 |
| `LV3` | 0 | 0 | 0 | 8 | 16 | 20 | 24 |
| `LV4` | 0 | 0 | 0 | 8 | 16 | 20 | 24 |
| `LV5` | 0 | 0 | 0 | 6 | 10 | 16 | 20 |
| `LV6` | 0 | 0 | 0 | 6 | 10 | 16 | 20 |
| `SCAT` | 0 | 0 | 0 | 20 | 40 | 60 | 100 |

Standard ways symbols use left-to-right consecutive reel counts. `SCAT` row is used for scatter pays. Base-game `SCAT` pays anywhere per Planner, while Free Spins `SCAT` evaluation remains ambiguous between anywhere and left-to-right and must remain configurable or explicitly reported in Coder outputs.

## Function Contracts

### generatePayWindow

Purpose: construct a visible row-major `[4x6]` pay window from one six-reel strip set.

Inputs:

| Name | Type | Shape |
|---|---|---|
| `ReelSet` | `Symbol reelset` | `[6][variable]` |
| `ReelSize` | `std::vector<int>` | `[6]` |
| `rng` | RNG source/state | `[1]` |

Outputs:

| Name | Type | Shape |
|---|---|---|
| `pay_window` | `std::vector<std::vector<Symbol>>` | `[4x6]` |

Global/probTable dependencies: `NO_OF_ROWS`, `NO_OF_REELS`.

Shape contract: `ReelSet` has six reels; `ReelSize` has six lengths; output has four rows and six columns.

Pseudocode:

```text
for reel_index from 0 to 5:
  sample stop_index uniformly from 0 to ReelSize[reel_index] - 1
  for row_index from 0 to 3:
    source_index = (stop_index + row_index) modulo ReelSize[reel_index]
    pay_window[row_index][reel_index] = ReelSet[reel_index][source_index]
return pay_window
```

Invariants: no symbols are created or transformed; wrap-around uses reel length; output is always `[4x6]`.

Trace emission requirements: emit `ReelSet`, `ReelSize`, `NO_OF_ROWS`, `NO_OF_REELS`, and `pay_window`.

### additionalCOR

Purpose: optionally inject additional base-game `COR` symbols into an existing pay window.

Inputs:

| Name | Type | Shape |
|---|---|---|
| `pay_window` | `std::vector<std::vector<Symbol>>` inout | `[4x6]` |

Outputs:

| Name | Type | Shape |
|---|---|---|
| `pay_window` | `std::vector<std::vector<Symbol>>` | `[4x6]` |

Global/probTable dependencies: `probTable.corTrigger`, `probTable.numCorWeight`, `probTable.baseCORValueWeights`.

Shape contract: input and output window remain `[4x6]`; number of added `COR` symbols is at most sampled count and at most eligible cells.

Pseudocode:

```text
sample trigger from probTable.corTrigger
if trigger is No:
  return pay_window unchanged
sample target_cor_count from probTable.numCorWeight
eligible_cells = every cell whose current symbol is not COLLECT, WILD, or SCAT

for each selected cell:
  replace symbol with COR
  assign COR xBet value using probTable.baseCORValueWeights
return pay_window
```

Invariants: never overwrite `COLLECT`, `WILD`, or `SCAT`; never resize the window; do not inject in Free Spins.

Trace emission requirements: emit input/output `pay_window`, `probTable.corTrigger`, and `probTable.numCorWeight`. `baseCORValueWeights` must be emitted when COR values are assigned.

### waysWinCalculation

Purpose: evaluate standard 4096-ways wins and scatter wins without Free Spins WILD multipliers.

Inputs:

| Name | Type | Shape |
|---|---|---|
| `pay_window` | `std::vector<std::vector<Symbol>>` | `[4x6]` |

Outputs:

| Name | Type | Shape |
|---|---|---|
| `win` | `int` or win total field | `[1]` |
| `WinBreakdown` | struct containing `ways_win`, `scatter_win`, `scatter_count`, `symbol_hits` | `[1]` |

Global/probTable dependencies: `PayTable` matrix view `[12x7]`, `STANDARD_SYMBOL_COUNT`, canonical standard symbol list, `WILD` substitution rules, configured `SCAT` evaluation rule.

Shape contract: reads `[4x6]`; paytable rows use 12 payable symbols; paytable columns use counts 0 through 6.

Pseudocode:

```text
ways_win = 0
for each standard_symbol in HV1..HV5, LV1..LV6:
  reel_match_counts = empty
  for reel_index from 0 to 5:
    count matches where cell is standard_symbol or WILD
    if count is 0:
      stop scanning reels for this symbol
    append count to reel_match_counts
  symbol_count = length(reel_match_counts)
  if PayTable[standard_symbol][symbol_count] > 0:
    ways = product of reel_match_counts
    ways_win += PayTable[standard_symbol][symbol_count] * ways
scatter_count = count SCAT according to active mode
scatter_win = PayTable[SCAT][scatter_count]
return WinBreakdown
```

Invariants: `WILD` substitutes only for standard symbols; `SCAT`, `COR`, and `COLLECT` never substitute; no Collect payout is resolved here.

Trace emission requirements: emit `pay_window`, `PayTable`, `STANDARD_SYMBOL_COUNT`, and `win`.

Pseudocode:

```text
remaining_spins = num_of_free_spins

  pay_window = generatePayWindow(FG1_Reels, FG1_ReelSize)
  breakdown = waysWinCalculationWithMultiplier(pay_window)
  session_win += breakdown.ways_win + breakdown.scatter_win
  scatter_count = breakdown.scatter_count

    remaining_spins += retrigger award for scatter_count
  remaining_spins -= 1
return FeatureResult
```

### FreeGameFeature_2

Purpose: run Blitz Spins. The name is retained for trace compatibility; this is not Free Spins.

Inputs:

| Name | Type | Shape |
|---|---|---|
| `num_of_blitz_spins` | `int` | `[1]` |
| `rng` | RNG source/state | `[1]` |

Outputs:

| Name | Type | Shape |
|---|---|---|
| `FeatureResult` | struct containing `session_win` and spins played | `[1]` |

Global/probTable dependencies: `probTable.freeGamenumCollect`, `probTable.freeGameCOROccur`, `probTable.goodCORTable`, `probTable.badCORTable`, `probTable.CORValues`.

Shape contract: every Blitz window is `[4x6]`.

Pseudocode:

```text
session_win = 0
for each blitz spin from 1 to num_of_blitz_spins:
  create empty 4x6 Blitz window
  sample collect_count from probTable.freeGamenumCollect
  place collect_count COLLECT symbols on reels 1 and/or 6, at most one per reel
  for every non-COLLECT cell:
    sample whether COR appears using probTable.freeGameCOROccur
  if no COR appears:
    force one eligible non-COLLECT cell to COR
  choose exactly one COR to use probTable.goodCORTable over probTable.CORValues
  assign every other COR using probTable.badCORTable over probTable.CORValues
  collect_result = resolveCollect(blitz_window, BASE_BET_FIXED_COINS)
  session_win += collect_result.collect_win
return FeatureResult
```

Invariants: only `COR`, jackpot-valued `COR`, and `COLLECT` appear; every Blitz spin has at least one `COLLECT` and at least one `COR`; no retriggers.

Trace emission requirements: emit `num_of_blitz_spins`, all Blitz probability tables, nested `resolveCollect`, and `FeatureResult`.

### runOneSpin

Purpose: execute one paid base-game round and selected triggered feature session.

Inputs:

| Name | Type | Shape |
|---|---|---|
| `bet_fixed_coins` | `int` | `[1]` |
| `feature_choice` | enum `NONE`, `FREE_SPINS`, `BLITZ_SPINS` | `[1]` |
| `rng` | RNG source/state | `[1]` |

Outputs:

| Name | Type | Shape |
|---|---|---|
| `SpinResult` | struct containing base win, collect win, feature win, total win | `[1]` |

Global/probTable dependencies: all base reelsets and sizes, `probTable.ReelsetWeights`, feature award tables, function dependencies above.

Shape contract: base and feature windows are `[4x6]`; reel size vectors are `[6]`; paytable view is `[12x7]`.

Pseudocode:

```text
select base reelset using probTable.ReelsetWeights
if B1 selected:
  pay_window = generatePayWindow(BG1_Reels, BG1_ReelSize)
else:
  pay_window = generatePayWindow(BG2_Reels, BG2_ReelSize)
pay_window = additionalCOR(pay_window)
base_breakdown = waysWinCalculation(pay_window)
collect_result = resolveCollect(pay_window, bet_fixed_coins)
feature_win = 0
if base_breakdown.scatter_count >= 3:
  if feature_choice is FREE_SPINS:
    award = Free Spins award for scatter_count
    feature_result = FreeGameFeature_1(award)
    feature_win = feature_result.session_win
  if feature_choice is BLITZ_SPINS:
    award = Blitz Spins award for scatter_count
    feature_result = FreeGameFeature_2(award)
    feature_win = feature_result.session_win
total_win = base_breakdown.ways_win + base_breakdown.scatter_win + collect_result.collect_win + feature_win
return SpinResult
```

Invariants: base feature trigger is 3+ `SCAT`; Collect resolves before feature handling; do not change base bet; do not auto-select feature choice unless caller/test explicitly provides it.

Trace emission requirements: emit `probTable.ReelsetWeights`, nested calls in order, and final `SpinResult.total_win`.

## Human-Readable Execution Explanation

`runOneSpin` starts by selecting `BG1_Reels` or `BG2_Reels` from `probTable.ReelsetWeights`, then builds a `[4x6]` base window. Base additional COR is optionally injected using `corTrigger`, `numCorWeight`, and `baseCORValueWeights`, while preserving the rule that added COR cannot overwrite `COLLECT`, `WILD`, or `SCAT`.

The base window is then evaluated for standard 4096-ways wins and scatter wins. Collect is resolved after ways/scatter evaluation and before feature handling. If fewer than three base `SCAT` symbols are present, the spin returns only base, scatter, and collect wins.

If three or more base `SCAT` symbols appear, the caller-provided feature choice determines whether to run Free Spins (`FreeGameFeature_1`) or Blitz Spins (`FreeGameFeature_2`). Free Spins use `FG1_Reels`, apply all visible WILD multipliers to ways wins, and support retriggers. Blitz Spins generate COR/COLLECT-only windows, guarantee at least one COLLECT and one COR, assign one Good-table COR value per spin and Bad-table values to all other CORs, then resolve Collect.
